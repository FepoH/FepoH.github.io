This directory contains sound files for deployment as mintty bell sounds.

Mintty bell sounds

The BOING sound
is available on YouTube, BOING.WAV.

Other nice boing sounds could be extracted from
YouTube, Boing 0
or
YouTube, Boing 2.

Further sounds are selected from a contribution of “naknak”, with some instructions
how to make bell sounds
from sound sources that may be useful to others.
The sounds are provided under the creative commons license.
See the subfolder archive in the mintty repository for the complete set.

Suitable Windows sounds may be Windows Default.wav, Windows Ding.wav,
Windows Critical Stop.wav, from the Windows subfolder Media.

To deploy a sound file to be listed in the Options dialog,
simply copy a .wav file into the config subfolder sounds
(e.g. in $HOME/.config/mintty or $APPDATA/mintty).

google编程规范

头文件

  • #define保护
  • 尽量使用前置申明，减少#include的数量（能依赖声明就不要依赖定义）
  • 只有当函数只有10行或更少时才定义内联函数
  • 定义函数时，参数顺序为：输入参数在前，输出参数在后（不强制要求，难以遵守）
  • 源文件中头文件的包含次序：
    • 源文件的头文件
    • C系统文件
    • C++系统文件
    • 其他库头文件
    • 本项目内头文件

类

  • 构造函数
    1. 一般只进行无意义的成员变量初始化，其他有意义的成员变量放入init()函数中
    2. 使用一个成员标记对象是否已经初始化成功
  • 默认构造函数
    1. 如果没有其他构造函数，并且类定义了成员变量，定义一个默认构造函数，初始化成员变量，使调试变得更简单。
  • 对单个参数的构造函数使用关键词explicit，避免隐式转换
  • 拷贝构造函数：仅在需要拷贝的类对象使用拷贝构造函数，否则要使用noncopyable
  • 容器中应该尽量使用指向指针，而不是定义拷贝构造和赋值构造
  • 仅当只有数据时使用struct（可包含构造，析构，init，reset，valid等对数据处理的函数），其他情况一律考虑使用class
  • 继承：使用组合通常比继承更合适，如果使用继承，只是用public继承，努力做到在在“is a”的情况系才使用继承。
    • 组合 > 实现继承 > 接口继承 > 私有继承
  • 除了少数特定环境外，不要定义重载操作符，在stl容器内，应该尽量使用仿函数，而不是重载运算符
  • 数据成员私有化，定义getXX()和setXX()获取和设置数据成员
  • 声明次序：
    • typedef和enum
    • 常量
    • 构造函数
    • 析构函数
    • 成员函数（包含静态成员函数）
    • 数据成员（包含静态数据成员）
  • .cc中函数的定义应该尽量与声明一致
  • 倾向于编写短小、凝聚的函数，如果函数超过40行，考虑分割函数。

命名规范

  • 文件名全部采用小写，可使用-和_ http_server_log.cc
  • c++文件以.cc为后缀
  • 类、结构体、类型定义(typedef)、枚举：每个单词以大写字母开头，不包含下划线
  • 变量名：
    • 一律小写，单词间以下划线相连
    • 类成员变量以下划线结尾 my_variable_
    • 结构提成员结尾不用下划线 my_variable
  • 常量名（局部，全局，类中）：
    • 在名称前加k，后接大写字母开头的单词 kRecvBuffer
  • 函数名：
    • 普通函数：大小写混合 AddTimer
    • 存储函数：与变量名匹配 my_variable–>my_variable set_my_variable
  • 命名空间：全是小写
  • 枚举命名，宏命名：全部大写，单词之间采用下划线相连。
